## Flujo de trabajo con WSL y Docker

Gabriel prefiere trabajar con WSL2 y Docker.

Buenas prácticas deseadas:

- Guardar proyectos dentro del home de WSL, no en `/mnt/c`, por rendimiento.
- Abrir VS Code desde WSL con `code .`.
- Usar Docker Compose para dev.
- Montar código como volumen en desarrollo.
- Usar `uvicorn --reload` en desarrollo.
- Separar Docker Compose dev y prod.

Estructura común preferida:

```txt
project/
├── app/
├── docker/
│   ├── dev/
│   │   └── docker-compose.dev.yml
│   └── prod/
│       └── docker-compose.prod.yml
├── .env.dev
├── .env.prod
└── README.md
```
